# StreetComb — Vision

## High concept
StreetComb is a mobile-first fighting game where the player performs movement, attacks, and special combos through finger gestures drawn directly on the screen.

The game is not a classic Street Fighter clone with worse virtual buttons. It is a touch-native combat game built around fluid gesture input, strong visual feedback, and distinct fighter identities.

## Core fantasy
You do not press buttons. You draw the fight.

## Product pillars
1. Touch-first combat
2. Fast, readable, satisfying fights
3. Easy to start, hard to master
4. Distinct fighters with their own combat grammar
5. Arcade progression with tournament structure

## Vertical slice goal
Validate one critical question:

Can gesture-based fighting feel responsive, readable, and fun on mobile?

## Vertical slice content
- 2 playable fighters
- 1 arena
- 1 short tournament run
- 6-8 reliable gestures
- Basic HUD
- Enemy AI
- Victory/defeat flow
- Minimal between-fight upgrade choice

## First two fighters
### Raze
- Fast, aggressive, accessible
- Strong dashes and pressure
- Electric / urban visual language

### Iron Monk
- Heavier, technical, defensive
- Strong punish tools and charged attacks
- Disciplined / impactful visual language

## Recommended game structure
Short tournament runs with light roguelite progression:
- 3 fights per run in slice
- upgrades between fights
- longer content later through unlocks, characters, modes, and difficulty

## Primary design risks
1. Gesture recognition feels inconsistent
2. Combat lacks impact and responsiveness
3. Visual clarity breaks on mobile
4. Scope expands into a full traditional fighter too early

## Production philosophy
Build the heart first:
- input
- combat feel
- feedback
- character identity

Do not expand roster/content until the core fight loop feels genuinely good.
